# github copilotでトークン数を減らす方法についてレポートを作って

**作成日**: 2026-05-23
**調査テーマ**: GitHub Copilotのトークン消費を削減するための方法

---

## エグゼクティブサマリー

GitHub Copilotのトークン消費削減は、コンテキストの絞り込み・プロンプト設計・チャット履歴管理の三本柱で構成される。`.copilotignore`でビルド成果物や依存関係ファイルを除外し、`copilot-instructions.md`で共通指示を一元化することが最初の一手として効果が高い。日常操作では「新トピック＝新スレッド」のルールとTab補完優先を習慣化することで、月次クォータを効率よく使い切れる。

## 背景・目的

GitHub Copilotは月ごとのトークン上限またはクレジット制限のもとで動作する。個人プランでは上限が低めに設定されており、無駄なトークン消費を防いで限られたクォータを効率的に活用することが重要になる。

## トークンとコンテキストウィンドウの仕組み

GitHub Copilotはリクエストごとにコンテキストをトークンとして送信する。消費が増える主な原因は以下の通り。

- 開いているファイルやタブが多い → 自動的にコンテキストへ含まれる
- チャット履歴が長い → 毎回の送信量が積み上がる
- `@workspace`を多用する → プロジェクト全体がインデックス対象になる
- `node_modules/`等を除外していない → 参照対象が膨大になる

## コンテキストの絞り込み

### 不要なタブを閉じる

Copilot Chatは現在開いているファイルをコンテキストへ自動的に含める。作業と無関係なタブはこまめに閉じる。

### `.copilotignore`を設定する

`.gitignore`と同書式のファイルをリポジトリルートに配置する。

```
node_modules/
dist/
build/
*.log
*.lock
coverage/
```

ビルド成果物・依存関係・ログを除外するだけで効果が大きい。

### `#file:`で特定ファイルを参照する

`@workspace`の代わりに`#file:src/foo.ts`のように特定ファイルを参照すると、コンテキストの広がりを抑えられる。

## プロンプト設計

### 簡潔・具体的に書く

曖昧な質問は追加のやり取りを生み、消費トークンが増える。

- 悪い例：「このコードのバグを探して」
- 良い例：「XX行目の関数でYYが起きる原因と修正方法を教えて、説明は不要でコードのみ出力して」

### `copilot-instructions.md`で共通指示を一元化する

`.github/copilot-instructions.md`にプロジェクト共通のルールを書いておくと、毎回プロンプトに書く必要がなくなる。ただしこのファイル自体もトークンを消費するため、簡潔に保つ。

```markdown
# Copilot Instructions
- TypeScriptを使用。`any`型は禁止
- コメントは日本語で
- テストはVitest
```

## チャット履歴の管理

Copilot Chatは会話履歴全体を毎回送信する。1つのスレッドを長く使い続けるとトークンが積み上がる。

- 新しいトピックに移る際は新しい会話スレッドを開始する
- 前の会話を参照する必要がなければ新スレッドが効率的

## インライン補完（Tab補完）の優先使用

CopilotのTab補完（Ghost Text）はChat APIより送信コンテキストが小さく、トークン効率が高い。

| 用途 | 推奨手段 |
|------|---------|
| 関数・変数名の補完 | Tab補完 |
| 短いコードブロック | Tab補完 |
| 設計の相談・説明 | Chat |
| 複雑なリファクタリング | Chat |

## モデル選択の最適化

VSCodeのCopilot設定でモデルを選択できる場合、タスクに応じて使い分ける。

| タスク | 推奨モデル |
|--------|----------|
| 単純な補完・リファクタリング | 軽量モデル |
| 複雑な設計・大規模な説明 | 高性能モデル |

## 結論・提言

優先度の高い順に実施する：

1. **`.copilotignore`を設定する**（一度設定すれば恒久的に効く、高効果）
2. **チャット履歴をこまめにリセットする**（長時間作業で特に重要）
3. **Tab補完をChatより優先する**（シンプルな補完はChatを使わない）
4. **プロンプトを簡潔・具体的にする**（習慣として定着すれば継続的に効く）
5. **`copilot-instructions.md`で共通指示を一元化する**（繰り返しの入力を削減）
