# Copilot Chat の会話履歴リセットによるトークン節約

調査日: 2026-05-24

## 概要

Copilot Chat は各リクエスト時に**それまでの会話履歴をまるごと送信**する仕組みのため、会話が長くなるほど1リクエストあたりの入力トークンが膨れ上がる。これをリセットすることでトークン節約になる。

---

## 会話履歴の送信の仕組み

- Copilot Chat は LLM ベースのチャットシステム共通の仕組みとして、リクエストごとに過去のやり取り（会話ターン）を全てプロンプトに含めて送信する
- 会話が進むほど入力トークンが増加し、2026年6月からの従量課金ではそれがそのままコストに直結する
- コードスニペットやファイルが会話に含まれていると、その分もトークンとして蓄積される

## /compact・/clear コマンドについて

出典: https://code.visualstudio.com/docs/copilot/reference/copilot-vscode-features

VS Code の Copilot Chat には以下のコマンドが存在する。

- **`/compact`** — 会話コンテキストを要約して圧縮する。コンテキストウィンドウが埋まってきたときに有効
- **`/clear`** — 新しいチャットセッションを開始する（履歴をまるごとリセット）

`/compact` は会話履歴を要約に置き換えることで入力トークンを削減しつつ文脈を保持する。`/clear` は完全リセット。

## 2026年6月課金制度との関係

- 6月からはプレミアムモデル（Sonnet 等）の入力トークンも従量課金対象となる見込み
- 会話を引き延ばすほど後半のリクエストが高コストになるため、こまめなリセットが直接的なコスト削減になる

## 推奨プラクティス

1. トピックが変わったら新規チャットを開始する
2. 解決済みの話題を続ける必要がないなら迷わずリセット
3. 引き継ぎたい情報がある場合は「これまでの結論：○○」と冒頭に貼り付けてから新チャット開始
