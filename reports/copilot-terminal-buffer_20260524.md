# @terminal のターミナルバッファとトークン消費

作成日: 2026-05-24
参照issue:
- https://github.com/microsoft/vscode/issues/251088
- https://github.com/microsoft/vscode/issues/280366

---

## 概要

VS Code の Copilot Chat で `@terminal` を使うと、ターミナルの出力内容が Copilot に送信される。「どこまでの出力を読むか」はシェルインテグレーションのマーカー次第で、設定によってはトークンが大量消費される動作がある。

---

## シェルインテグレーションによる読み取り範囲の決定

`@terminal` はシェルインテグレーション（shell integration）のマーカーを使って読み取り範囲を決定する。VS Code がターミナル内のコマンド実行位置を記録する仕組みで、コマンドの開始・終了位置がターミナルバッファ上にマーキングされる。

**通常の動作:** マーカーを参照し「最後に実行したコマンドの出力」のみを読み取る。トークン消費は最小限。

---

## フォールバック動作（問題の本質）

出力が多くスクロールバック上限を超えた場合、古いマーカーはバッファから消える。マーカーが消えた状態で `@terminal` を使うと、**バッファに残っている最古の行から現在行まで全体**を読みに行くフォールバック動作になる。

バッファに大量の行が残っていると、その全行分のテキストがコンテキストとして送信される。

---

## 実害の報告

スクロールバックを 100,000 行に設定した状態で Copilot Chat を使うと、Copilot 拡張のメモリ使用量が急増し VS Code がクラッシュするという issue が実際に報告されている。Microsoft の対応は「**as-designed（設計通り）**」で閉じられており、バグではないとされている。

---

## デフォルト設定

VS Code のターミナルスクロールバック行数のデフォルト値は `terminal.integrated.scrollback` = **1000行**。

---

## 回避策

**使う前に `clear` する**
ターミナルをクリアしてからクエリを投げることで、フォールバック時の送信量を抑えられる。

**スクロールバック設定を大きくしすぎない**
デフォルト（1000行）から必要以上に増やさない。100,000行などの極端な値はクラッシュの原因になりうる。

---

## まとめ

| 状況 | 動作 | トークン消費 |
|---|---|---|
| マーカーあり（通常） | 最後のコマンド出力のみ送信 | 少ない |
| マーカーなし（フォールバック） | バッファ全体を送信 | バッファ行数に比例 |
