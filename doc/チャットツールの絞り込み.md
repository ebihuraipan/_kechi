# チャットツールの絞り込み

エージェントモードで有効にしているツールは、使用しなくてもその定義がリクエストのコンテキストに乗り続ける。不要なツールをオフにするだけで毎リクエストのトークンを削減できる。

## 設定場所

チャット入力欄のツールアイコン（クリップアイコン横）→「チャットに使用できるツールを選択してください」

## 組み込みツール一覧と推奨設定

「エージェントでバッチファイルを実行してファイル・フォルダ操作する」用途の場合：

| ツール | 子ツール数 | 推奨 | 理由 |
|--------|-----------|------|------|
| agent | 1 | **オフ** | サブエージェント委任は不要 |
| browser | 10 | **オフ** | ブラウザ自動化は不要 |
| execute | 8 | 残す | ターミナル・タスク実行の中核 |
| new | 0 | **オフ** | スキャフォールドは稀 |
| read | 3 | 残す | タスク・ターミナル出力の読み取り |
| todo | 0 | **オフ** | Copilot に ToDo 管理させないなら不要 |
| vscode | 2 | 残す | VS Code 基本機能 |

オフにすると `browser`(10) + `agent`(1) = **11個分のツール定義**がコンテキストから除外される。

## 注意

- MCP サーバー経由のツールも同様。接続しているだけで全ツール定義がコンテキストに乗る。
- 設定はチャットセッションをまたいで保持される。
- 個別の子ツール単位でもオン/オフ可能。
