## プロンプト設計

曖昧な質問は追加のやり取りを生み、消費トークンが増える。

- 悪い例：「このコードのバグを探して」
- 良い例：「XX 行目の関数で YY が起きる原因と修正方法を教えて、説明は不要でコードのみ出力して」

`.github/copilot-instructions.md` にプロジェクト共通のルールを書いておくと、毎回プロンプトに書く必要がなくなる。ただしこのファイル自体もトークンを消費するため簡潔に保つ。

```markdown
# Copilot Instructions
- TypeScript を使用。`any` 型は禁止
- コメントは日本語で
- テストは Vitest
```

効果★★☆
