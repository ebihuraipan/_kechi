## 指示書を「出しっぱなし」にしない（applyTo）

`.github/instructions/*.instructions.md` → **特定ファイルを触っているときだけ**読まれる

フロントマターでファイルの先頭に `---` で囲んだ部分を書き、`applyTo:` に対象ファイルのパターンを指定する。ファイル名は必ず `.instructions.md` で終わること（例: `frontend.instructions.md`）。

```markdown
applyTo: "**/*.ts"
```

パターンの例:

| 書き方 | 対象 |
|---|---|
| `"**/*.rb"` | 全Rubyファイル |
| `"**/*.ts,**/*.tsx"` | TypeScript系（カンマで複数指定） |
| `"**/*.test.*"` | テストファイル全般 |
| `"db/**"` | dbフォルダ以下 |

効果★★★
