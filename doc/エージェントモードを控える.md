## エージェントモードを控える

vscodeのAIチャット欄ではask,edit,agentのモードがある。
基本全部 agentモードで使えば機能的には問題ない、が、

agentモード は自律的にファイルを読み・探索するため、単一ファイルへの指示でも**プロジェクト構造や関連ファイルを自動で参照**し、コンテキストとトークンが膨らむ。
関連ファイルの読み込み・参照・整合性チェックのために、1 回の応答に乗せるコンテキスト量が **Ask/Edit に比べて桁違いに大きい**。
agentモード の内部動作は「計画 → 実行 → 検証 → 修正」のループ構造になっている。各ループでシステムプロンプト・タスク説明・ツール定義・実行履歴がすべてコンテキストに積み上がっていく。これが繰り返されるほど、1 タスクあたりの総トークン数は膨らむ。

```
質問・調べもの？ → Ask Mode（最小コスト）
変更先ファイルが明確で局所的？ → Edit Mode（コスト小、制御しやすい）
複数ファイル横断・自律実行が必要？ → agentモード（コスト大、パワフル）
```

タスクの複雑さに合わせてモードを下げる。「agentモードで出来るから」という理由だけで毎回 Agent を使わず、Ask → Edit → Agent の順で必要性を確認してから選ぶ。

効果★★★
