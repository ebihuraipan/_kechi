## マルチルートワークスペースという選択肢

**基本は個別プロジェクト単体で開く**（Copilot トークン節約最優先）

VSCode の `.code-workspace` ファイルを使うと、複数プロジェクトを選択的に追加しつつ各フォルダ配下のみをインデックスできる。

```jsonc
// mywork.code-workspace
{
  "folders": [
    { "path": "project-a" },
    { "path": "project-b" }
  ]
}
```

| 開き方 | トークン節約 | 利便性 |
|--------|------------|--------|
| git ルート全体 | 低 | 高 |
| .code-workspace（一部） | 中 | 中 |
| 個別プロジェクト単体 | 高 | 低 |

効果★★☆
