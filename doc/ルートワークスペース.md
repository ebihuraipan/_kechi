## git ルートで開く場合の影響

1つの git リポジトリに複数プロジェクトが含まれる構成（モノレポ等）で、git ルート（`/repo-root/`）ごと VS Code で開くと：

- Copilot はルート以下の全ファイルを参照対象とするため、作業中のプロジェクトと無関係なファイルがコンテキストに混入する
- インデックスが肥大化し Copilot の推論精度が下がる場合がある
- `.copilotignore` でプロジェクトごとの除外を書く必要があり管理が煩雑

**対策**：作業するプロジェクト単体（例: `/repo-root/project-a/`）を VS Code で開くと、Copilot のコンテキスト範囲がそのプロジェクトだけに絞られる。複数プロジェクトをまたぐ場合は `.code-workspace` で必要なフォルダのみ追加する。

効果★★☆
